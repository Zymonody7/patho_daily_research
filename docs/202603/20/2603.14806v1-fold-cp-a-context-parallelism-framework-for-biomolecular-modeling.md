## 摘要
理解细胞机制需要对大型生物分子组装体进行原子尺度的重建。然而，预测这些系统的结构一直受到 AlphaFold 3 等模型对硬件显存需求的限制，导致单张 GPU 能够处理的残基数量存在仅几千个的实际上限。在此，我们介绍了 NVIDIA BioNeMo Fold-CP，这是一个上下文并行框架，通过将协同折叠模型的推理和训练流水线分布到多个 GPU 上来克服这一障碍。我们以 Boltz 模型作为开源参考架构，并实现了自定义的多维原语，从而有效地并行化了密集三角形更新以及窗口批处理局部注意力的不规则、数据依赖模式。我们的方法实现了高效的显存扩展；对于分布在 P 个 GPU 上的 N 个 token 输入，单设备显存占用按 O(N^2/P) 比例缩放，使得在 64 个 NVIDIA B300 GPU 上能够预测超过 30,000 个残基的组装体结构。我们通过成功的开发者用例展示了该方法的科学实用性：Fold-CP 实现了对哺乳动物蛋白复合物综合资源 (CORUM) 数据库中 90% 以上数据的评分，并完成了与内在无序区域结合的疾病相关 PI4KA 脂质激酶复合物的折叠，且无需进行裁剪。通过为具有完整全局上下文的大规模系统建模提供可扩展的路径，Fold-CP 代表了迈向实现虚拟细胞的重要一步。

## Abstract
Understanding cellular machinery requires atomic-scale reconstruction of large biomolecular assemblies. However, predicting the structures of these systems has been constrained by hardware memory requirements of models like AlphaFold 3, imposing a practical ceiling of a few thousand residues that can be processed on a single GPU. Here we present NVIDIA BioNeMo Fold-CP, a context parallelism framework that overcomes this barrier by distributing the inference and training pipelines of co-folding models across multiple GPUs. We use the Boltz models as open source reference architectures and implement custom multidimensional primitives that efficiently parallelize both the dense triangular updates and the irregular, data-dependent pattern of window-batched local attention. Our approach achieves efficient memory scaling; for an N-token input distributed across P GPUs, per-device memory scales as $O(N^2/P)$, enabling the structure prediction of assemblies exceeding 30,000 residues on 64 NVIDIA B300 GPUs. We demonstrate the scientific utility of this approach through successful developer use cases: Fold-CP enabled the scoring of over 90% of Comprehensive Resource of Mammalian protein complexes (CORUM) database, as well as folding of disease-relevant PI4KA lipid kinase complex bound to an intrinsically disordered region without cropping. By providing a scalable pathway for modeling massive systems with full global context, Fold-CP represents a significant step toward the realization of a virtual cell.

### 1. 核心问题：突破蛋白质折叠的“显存墙”
目前的顶级生物分子结构预测模型（如 AlphaFold 3 或开源的 Boltz-1/2）在处理大型复合物时面临严重的硬件瓶颈。
*   **数学矛盾**：模型需要维护一个“成对表示”（Pair Representation）矩阵，记录每两个氨基酸/原子之间的相互作用。这个矩阵的显存占用随序列长度 $N$ 呈平方级（$O(N^2)$）增长，计算量呈立方级（$O(N^3)$）增长。
*   **现实困境**：单张顶级 GPU（如 H100）通常只能处理约 2000-5000 个残基。但人体内许多关键的“细胞机器”（如病毒衣壳、大型信号转导复合物）动辄包含上万个残基。为了强行计算，研究者不得不对序列进行“裁剪”，这会导致模型丢失全局视野，预测结果支离破碎。

### 2. 白话版概述
想象你要处理一张巨大的 Excel 表格（蛋白质相互作用矩阵），单台电脑的内存塞不下。Fold-CP 的做法是将这张大表切成许多小方块（Tile），分给几十张显卡共同持有。它设计了一套精妙的“传球”机制（环形通信），让显卡在计算自己那块数据时，能高效地从其他显卡获取必要信息，而不需要任何一张显卡存储完整的表格。通过这种方式，它成功在 64 张显卡上预测了包含 3 万个残基的超大结构，这在以前是不可想象的。

---

### 3. 方法部分：多维并行与自定义原语
Fold-CP 的核心是**上下文并行（Context Parallelism, CP）**，针对生物建模的特殊几何运算进行了重构：

*   **2D 磁贴切分（2D Tiling）**：将 $N \times N$ 的成对表示矩阵切分为 $\sqrt{P} \times \sqrt{P}$ 的网格（$P$ 为 GPU 数量）。每张显卡只负责其中一个子块。
*   **自定义分布式算子**：
    *   **三角更新（Triangle Updates）**：这是 AlphaFold 类模型特有的算子，用于模拟几何一致性。Fold-CP 实现了分布式三角注意力（Row/Column Attention）和三角乘法，利用环形通信（Ring Communication）在不还原大矩阵的情况下完成计算。
    *   **窗口批处理（Window Batching）**：在原子层面，模型只关注空间邻域。Fold-CP 利用 Toeplitz 矩阵的数学特性，实现了无需分配巨大索引矩阵的局部注意力，大幅节省了显存。
*   **关键设计取舍**：
    *   **强制正方形拓扑**：要求 GPU 数量必须是平方数（如 4, 16, 64），以保证行、列通信的对称性和代码实现的简洁性。
    *   **放弃原生 DTensor**：为了防止 PyTorch 自动触发导致 OOM（显存溢出）的全量聚合操作，团队手动编写了所有的分布式 Autograd 函数。

### 4. 实验部分
*   **数据与任务**：使用 Boltz-1/2 作为基础模型，在 CORUM（哺乳动物蛋白复合物数据库）和 PDB（蛋白质数据库）上进行推理和训练测试。
*   **Baseline**：单卡推理（DP 模式）以及传统的序列裁剪方法。
*   **评价指标**：lDDT（局部距离差值测试，衡量结构准确度）、TM-score（全局相似度）、显存缩放曲线。
*   **主要结果**：
    *   **显存线性缩放**：单卡显存占用随 GPU 数量增加呈 $O(N^2/P)$ 下降。
    *   **规模突破**：在 64 张 B300 GPU 上成功预测了 **32,000+** 个 token 的结构。
    *   **科学发现**：完整预测了 PI4KA 脂质激酶复合物（3,605 个残基），发现了以前因裁剪而无法观察到的新型相互作用位点。

### 7. 局限与风险
*   **模型幻觉（Artifacts）**：论文坦诚，虽然计算上能跑通长序列，但由于模型是在短序列（如 384-768 token）上训练的，在预测超长序列时会出现“链重叠”或“聚成一团”的物理错误。
*   **数据匮乏**：目前 PDB 数据库中 85% 的结构都小于 1900 个残基，缺乏超大规模结构的真值数据来进一步微调模型。
*   **硬件依赖**：该框架高度依赖 NVIDIA 的 NVLink 高速互联，在带宽较低的集群上通信开销可能会抵消并行收益。
