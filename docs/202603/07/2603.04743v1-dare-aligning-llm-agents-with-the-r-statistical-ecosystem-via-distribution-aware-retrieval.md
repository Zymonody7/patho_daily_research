## 摘要
大语言模型 (LLM) 智能体可以自动化数据科学工作流，但由于 LLM 在统计知识和工具检索方面存在困难，许多在 R 中实现的严谨统计方法仍未得到充分利用。现有的检索增强方法侧重于函数级语义而忽略了数据分布，导致匹配效果欠佳。我们提出了 DARE（分布感知检索嵌入），这是一种轻量级、即插即用的检索模型，它将数据分布信息整合到函数表示中，用于 R 包检索。我们的主要贡献包括：(i) RPKB，一个源自 8,191 个高质量 CRAN 软件包的精选 R 软件包知识库；(ii) DARE，一种将分布特征与函数元数据融合以提高检索相关性的嵌入模型；以及 (iii) RCodingAgent，一个面向 R 的 LLM 智能体，用于可靠的 R 代码生成，并提供了一套统计分析任务，用于在现实分析场景中系统地评估 LLM 智能体。实验表明，DARE 的 NDCG@10 达到了 93.47%，在软件包检索方面比最先进的开源嵌入模型高出多达 17%，同时使用的参数量显著减少。将 DARE 集成到 RCodingAgent 中在下游分析任务上取得了显著提升。这项工作有助于缩小 LLM 自动化与成熟的 R 统计生态系统之间的差距。

## Abstract
Large Language Model (LLM) agents can automate data-science workflows, but many rigorous statistical methods implemented in R remain underused because LLMs struggle with statistical knowledge and tool retrieval. Existing retrieval-augmented approaches focus on function-level semantics and ignore data distribution, producing suboptimal matches. We propose DARE (Distribution-Aware Retrieval Embedding), a lightweight, plug-and-play retrieval model that incorporates data distribution information into function representations for R package retrieval. Our main contributions are: (i) RPKB, a curated R Package Knowledge Base derived from 8,191 high-quality CRAN packages; (ii) DARE, an embedding model that fuses distributional features with function metadata to improve retrieval relevance; and (iii) RCodingAgent, an R-oriented LLM agent for reliable R code generation and a suite of statistical analysis tasks for systematically evaluating LLM agents in realistic analytical scenarios. Empirically, DARE achieves an NDCG at 10 of 93.47%, outperforming state-of-the-art open-source embedding models by up to 17% on package retrieval while using substantially fewer parameters. Integrating DARE into RCodingAgent yields significant gains on downstream analysis tasks. This work helps narrow the gap between LLM automation and the mature R statistical ecosystem.

## 论文详细总结（自动生成）

这篇论文由香港理工大学团队发表，旨在解决 LLM 智能体在处理专业统计分析任务时“空有逻辑、乱用工具”的问题。

### 1. 解决的问题与研究价值
**核心问题**：LLM 智能体在自动化数据科学任务时，往往偏向 Python 而冷落 R 语言。即便使用 R，也经常出现函数名幻觉、参数误用，或无法根据数据特征（如高维、稀疏、非正态分布）选择最合适的统计方法。
**研究价值**：R 语言拥有数十年积累的严谨统计生态（CRAN）。如果能让 AI 准确调用这些专业工具，将极大提升自动化数据分析的科学性和可靠性，特别是在生物信息学、临床试验等对统计严谨性要求极高的领域。

### 2. 白话版概述
目前的 AI 找工具就像在图书馆只看书名（语义检索），但统计分析需要看“适用条件”。比如，同样是回归分析，数据是正态分布还是泊松分布，选用的 R 包完全不同。
这篇论文做三件事：
1.  **建库**：把 8000 多个 R 语言包整理成带“说明书”的知识库。
2.  **训练检索器**：教 AI 不仅看用户想要做什么，还要看用户的数据长什么样（分布特征），从而精准匹配 R 函数。
3.  **造智能体**：做了一个能查手册、写 R 代码、运行并自我纠错的机器人。

### 3. 方法部分：核心思想与设计
*   **核心思想：分布感知（Distribution-Aware）**。检索时不只匹配“我要做聚类”，而是匹配“我要对**高维、非高斯分布**的表格数据做聚类”。
*   **RPKB 知识库构建**：从 CRAN 抓取 8,191 个高质量包，利用 LLM（如 Grok）将非结构化的文档转化为结构化的 **Data Profile（数据画像）**，包含：数据模态、分布假设、维度、缺失值处理等。
*   **DARE 模型结构**：采用轻量级的 **Bi-Encoder（双编码器）** 架构（基于 MiniLM）。
    *   **输入端**：将“用户查询”与“数据画像”拼接。
    *   **目标端**：将“函数文档”与“函数适用的数据画像”拼接。
*   **训练流程**：使用对比学习（InfoNCE Loss），让模型在向量空间中拉近“需求+数据特征”与“正确函数+适用条件”的距离。
*   **关键取舍**：放弃了追求超大规模参数，仅使用 23M 参数的轻量模型，以保证在智能体多轮对话中的低延迟（3.7ms/次）。

### 4. 实验部分
*   **数据与任务**：
    *   **检索实验**：在 RPKB 测试集上对比 24.5 万条合成查询。
    *   **智能体实验**：设计了 16 个复杂的统计任务（如生存分析、混合效应模型、基因调控评分估计）。
*   **Baseline（对比基准）**：包括 BGE-M3、Snowflake Arctic、E5 等目前最强的开源嵌入模型。
*   **评价指标**：NDCG@10（排名质量）、Recall@1（首选准确率）、Success Rate（任务最终成功率）。
*   **主要结果**：
    *   **检索**：DARE 以极小的参数量，在 NDCG@10 上达到 93.47%，超过参数量大其 20 倍的模型（提升约 17%）。
    *   **任务成功率**：集成 DARE 后，RCodingAgent 在下游任务的成功率最高提升了 56.25%（例如 Grok-4.1-fast 从 18.75% 提升至 75%）。

### 5. 资源与算力
*   **训练资源**：使用单张 NVIDIA A100 (80GB) GPU。
*   **训练细节**：AdamW 优化器，100 个 Epoch，Batch Size 为 256。

### 6. 真正可信的贡献
1.  **证明了“数据特征”是统计工具检索的关键**：实验数据有力支撑了加入 Data Profile 后检索精度的飞跃。
2.  **轻量化优势**：证明了在特定垂直领域（R 统计），经过精细数据增强的小模型可以全方位碾压通用大模型。
3.  **RPKB 知识库**：为后续 R 语言智能体的研究提供了高质量的结构化数据集。

### 7. 局限与风险
